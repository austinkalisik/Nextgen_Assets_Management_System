Nextgen Assets Management System
System Structure Overview

ERD Reference
- Visual ERD file: graph.png
- Supporting ERD notes: wireframe/ERD - Nextgen Assets Management System.txt
- Generated ERD snapshot: wireframe/erd-current.txt

Application Type
- Laravel + ReactJS CRUD application
- Backend: Laravel
- Frontend: ReactJS
- Routing: React Router on the frontend with Laravel API endpoints on the backend
- Data layer: MySQL through Laravel models and migrations
- Build tool: Vite

Business Ownership
- Product Name: Nextgen Assets Management System
- Owned by: Nextgen Technology
- Website: https://nextgenpng.net/
- Usage Model: built for Nextgen Technology and can also be offered to other departments that want to use the software

Core System Purpose
- Track office assets and stock items
- Manage assignments and returns
- Monitor low stock and maintenance alerts
- Manage users, suppliers, categories, departments, and settings
- Preserve audit history and operational accountability

Architecture Summary
- Laravel handles authentication, API routes, database access, validation, settings, notifications, and business logic
- ReactJS handles the user interface, navigation, forms, tables, dashboards, and interactive workflows
- The application behaves like a single-page admin system backed by Laravel APIs

Current ERD Alignment Notes
- graph.png was regenerated from the current local database using php artisan generate:erd
- The generated ERD currently reflects 10 modeled operational entities
- The generated ERD includes depreciation fields on Item
- The generated ERD may also show legacy local columns such as assigned_department_id on Assignment and profile_photo_path on User
- These notes describe the application structure without contradicting the generated ERD

Main CRUD Modules
- Inventory / Items
- Assignments
- Suppliers
- Categories
- Departments
- Users
- Notifications
- Activity Logs
- Settings
- Profile

Main User Flow
1. User logs in
2. Dashboard shows system overview
3. Inventory stores and updates asset records
4. Assignments issue assets to receivers or departments
5. Returns bring stock back
6. Notifications and logs record important activity
7. Settings control branding and feature behavior
