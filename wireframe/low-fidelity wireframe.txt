System Overview
- Laravel backend with ReactJS frontend
- API-driven CRUD system for inventory, assignments, users, departments, receivers, reports, notifications, and settings
- Owned by Nextgen Technology
- Main live inventory entity is Item, not the legacy Asset placeholder

Current Navigation
- Operations: Dashboard, Inventory, Assignments, Suppliers, User Guide
- Administration: Categories, Departments, Receivers, Users, Notifications, Activity Logs, Settings, Profile

Current Business Logic
- Departments are the first grouping for people and assignment reports.
- Receivers belong to departments.
- Assigning an item uses a Department / Person Receiving selector so the department context is clear before the asset is given out.
- Inventory supports Bulk stock and Serialized asset tracking.
- Purchase Date and Depreciation Start Date use the same YYYY-MM-DD input pattern with calendar picker support.
- Activity Logs and Assignments exports must follow the active filters.
